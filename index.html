全体像を把握する

Webページが表示されるまで

  1. ユーザーがブラウザでURLを入力する
  2. サーバーからHTML、CSS、JavaScriptファイルが送られてくる
  3. ブラウザがこれらのファイルを読み込み、画面に表示する

この3つのファイルがWebページを構成している

3つの技術の役割

HTML

ページの「内容」と「構造」を担当。文章、画像、リンクなど、何を載せるかを決める。見出し、段落、リストなど、内容の構造を決める。

CSS

ページの「見た目」を担当。色、フォント、サイズを決める。レイアウト（配置）を決める。装飾やアニメーションを加える。

JavaScript

ページの「動き」を担当。ボタンをクリックしたときの処理。入力内容のチェック。ページ内容の動的な変更。

例え話で理解する

家に例えると
  • HTML = 骨組み・間取り（構造）
  • CSS = 壁紙・家具・インテリア（見た目）
  • JavaScript = 電気・水道・家電（機能・動き）
人間に例えると
  • HTML = 骨格・体の構造
  • CSS = 服装・髪型・メイク
  • JavaScript = 動作・行動

それぞれの関係を理解する

3つの技術の連携
  • HTMLが土台となり、その上にCSSとJavaScriptが働きかける
  • HTMLだけでもページは表示できる（ただし見た目は質素）
  • CSSはHTMLの要素を装飾する
  • JavaScriptはHTMLの要素を操作する
ファイルの関係
  • HTMLファイル（.html）がメインのファイル
  • CSSファイル（.css）をHTMLから読み込む
  • JavaScriptファイル（.js）をHTMLから読み込む
  • 1つのHTMLに複数のCSS、JSを読み込むことも可能

HTMLを理解する

HTMLとは何か

HyperText Markup Language の略

Webページの内容と構造を記述するための言語

「タグ」と呼ばれる記号で内容を囲んで構造を作る

ブラウザがHTMLを読み取り、画面に表示する

タグの基本

タグは < と > で囲まれた記号

開始タグ <p> と終了タグ </p> でペアになる

例：<p>これは段落です</p>

基本的なタグ

<html>
HTML文書全体を囲む
<head>
ページの情報（タイトル、読み込むファイルなど）
<body>
ページに表示される内容

<h1> ～ <h6>
見出しを作る。h1が最も大きく重要、h6が最も小さい。ページには通常h1は1つだけ使用。

<p>
段落を作る
<br>
改行（終了タグなし）
<strong>
重要なテキスト（太字）
<em>
強調テキスト（斜体）

<a href="URL">
リンクを作る。href属性にリンク先のURLを指定。
<img src="パス" alt="説明">
画像を表示する。alt属性に画像の説明を入れる（アクセシビリティのため必須）。

<ul>
箇条書きリスト（順序なし）
<ol>
番号付きリスト（順序あり）
<li>
リストの各項目
<div>
ブロック要素のグループ化（縦に並ぶ）
<span>
インライン要素のグループ化（横に並ぶ）

セマンティックHTML

意味のあるタグを使うことで、構造が明確になる

検索エンジンやスクリーンリーダーが内容を理解しやすくなる

主なセマンティックタグ

  • <header> ：ヘッダー領域
  • <nav> ：ナビゲーション
  • <main> ：メインコンテンツ
  • <article> ：独立した記事
  • <section> ：セクション
  • <footer> ：フッター領域

CSSを理解する

CSSとは何か

Cascading Style Sheets の略

HTMLで作った構造に見た目を与える言語

色、サイズ、配置、アニメーションなどを指定できる

HTMLと分離することで、管理しやすくなる

CSSの基本的な書き方

セレクタ {
  プロパティ: 値;
}
セレクタ
どの要素に適用するか（例：p, h1, .クラス名）
プロパティ
何を変えるか（例：color, font-size）
値
どのように変えるか（例：red, 16px）

CSSの適用方法

インラインスタイル

HTMLタグに直接style属性を書く

<p style="color: red;">

手軽だが、管理が大変になるため非推奨

styleタグ

<head>内に<style>タグを書く

1つのHTMLファイル内でのみ有効

小規模なページや学習用に便利

外部ファイル（推奨）

別ファイル（.css）に記述

<link rel="stylesheet" href="...">

複数のHTMLで同じCSSを使える

基本的なプロパティ

色・背景
color
文字の色
background-color
背景色
background-image
背景画像
フォント
font-size
文字サイズ（px, em, rem）
font-family
フォントの種類
font-weight
文字の太さ
余白
margin
要素の外側の余白
padding
要素の内側の余白
ボックスモデル

内側から：内容 → padding → border → margin

width, height
箱のサイズ
border
枠線

JavaScriptを理解する

JavaScriptとは何か

Webページに動きをつけるためのプログラミング言語

ブラウザ上で動作する（クライアントサイド）

ユーザーの操作に反応して、ページを動的に変更できる

JavaScriptでできること

ユーザー操作への反応
  • ボタンクリックで処理を実行
  • マウスオーバーで表示を変える
  • キーボード入力を検知
ページ内容の動的変更
  • テキストや画像を書き換える
  • 要素を追加・削除する
  • スタイル（CSS）を変更する
データの処理
  • 入力フォームの値をチェック
  • 計算処理
  • データの保存・読み込み
その他
  • アニメーションの制御
  • 外部データの取得（API連携）
  • 複雑なWebアプリケーション

基本的な文法

変数

データを保存する箱

let 変数名 = 値;

例：let message = "こんにちは";

関数

処理をまとめたもの

function 関数名() {
  処理
}
イベント

ユーザーの操作を検知する仕組み

クリック、マウス移動、キー入力など

[補足] フロントエンドとバックエンド

フロントエンドとは
  • ユーザーの目に見える部分
  • ブラウザ上で動作する
  • 使用技術：HTML、CSS、JavaScript
  • デザインや使いやすさ（UI/UX）を担当
バックエンドとは
  • ユーザーの目に見えない部分
  • サーバー上で動作する
  • 使用技術：PHP、Python、Ruby、Node.jsなど
  • データベース、ユーザー認証を担当
連携の例
  1. ユーザーがログインボタンを押す（フロントエンド）
  2. 入力されたID・パスワードをサーバーに送信
  3. サーバーでデータベースを確認（バックエンド）
  4. 結果をフロントエンドに返す
  5. 画面を更新してログイン完了を表示（フロントエンド）

この講座で扱う範囲

この講座ではフロントエンドのみを扱います。バックエンドに興味がある方は、別途学習が必要です。
